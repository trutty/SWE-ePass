Release notes for Kendo UI v2013.1.226

Framework

What's new

  • Include VSDOC for all suites in Kendo Complete and Kendo Complete for ASP.NET MVC
  • Remove the default JSON parameterMap from DataSource Java wrapper implementation

DataSource

What's new

  • Support for virtualization with grouping

What's fixed

  • OData tranposrt does not support filtering with "Does not contain" operator

Globalization

What's new

  • Expand culture's numberFormat in order to support globalize.js

What's fixed

  • kendo.toString hangs out when try to format number with "# yr." format
  • Numeric formatting rounds to the last zero placeholder in the decimal part incorrectly
  • kendo.parseDate parses incorrectly ISO8601 dates with timezones

HierarchicalDataSource

What's new

  • Breaking change: children datasources are now lazily initialized

MVVM

What's new

  • Support for setting <select> values which do not exist in the source binding.

  • RequireJS support

  • Multiple drag&drop issues in IE10 when zoomed.

Chart

What's new

  • Add negativeColor for bar and column series
  • Breaking change: Justified option should be true by default for the Area chart.
  • Add multiple tooltip functionality.
  • Add crosshair functionality.
  • Export kendo.dataviz.requestFrame(callback, delay) and make it overridable
  • x/yAxis type can now be inferred from min/max options as well as series values
  • Add scatter and categorical pan and zoom demos

What's fixed

  • Category axis is rendered on top for bar charts with no defined categories
  • Chart automatic width and height are unreliable in Chrome
  • Reduced memory usage in IE (unique hashtable keys leak memory in IE < 10)
  • Axis type is incorrectly inferred when a x/yAxis has a name and the series data is of type Date
  • dashType is not applied unless for line series unless width is set
  • Axis range might not be calculated correctly for the first axis if it is named
  • Rendered elements don't have IDs in IE to avoid memory leaks. Use kendo.dataviz.getElement(id) instead of document.getElementByID(id) for locating them.
  • Date strings as x/y values result in error even when x/yAxis type is "date"
  • Error when using date value/x/yAxis and zoom*/drag* events
  • Chart with no set height can disappear when the page is zoomed
  • Possible hang when the axis baseUnit is set to hours/minutes and contains DST transition
  • The chart can hang if DST changes in the date axis range
  • Navigator selection can become misaligned
  • Memory leaks on dispose
